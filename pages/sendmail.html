API - Sendmail

Simplifique o envio de e-mails em suas aplicações com a API Sendmail da LOO9. Nossa interface robusta e fácil de usar permite integrar funcionalidades de envio de e-mail de forma eficiente e segura.

Com a API Sendmail, você pode:

  • Enviar e-mails transacionais (notificações, confirmações, etc.).
  • Implementar funcionalidades de e-mail marketing.
  • Automatizar o envio de mensagens personalizadas.
  • Integrar com diversos sistemas e plataformas.

Parâmetros Requeridos - Header:

  • Authorization: Sua chave de API (API Key) para autenticação. Exemplo: Bearer SUA_CHAVE_API

Parâmetros Requeridos - Body (Formato JSON):

  • smtp_server: Endereço do servidor SMTP. Exemplo: smtp.seudominio.com
  • smtp_port: Porta do servidor SMTP. Exemplo: 587
  • smtp_secure: Indica se a conexão é segura (TLS/SSL). Valores: true ou false.
  • email_from: Endereço de e-mail do remetente. Exemplo: seu_email@seudominio.com
  • password: Senha da conta de e-mail do remetente.
  • email_to: Endereço de e-mail do destinatário. Exemplo: destinatario@outrodominio.com
  • subject: Assunto do e-mail.
  • message: Corpo da mensagem (pode ser texto simples ou HTML).

Exemplo de Requisição (cURL):

curl -X POST \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer SUA_CHAVE_API" \
    -d '{
        "smtp_server": "smtp.seudominio.com",
        "smtp_port": 587,
        "smtp_secure": true,
        "email_from": "seu_email@seudominio.com",
        "password": "sua_senha",
        "email_to": "destinatario@outrodominio.com",
        "subject": "Assunto do E-mail",
        "message": "Conteúdo do e-mail em texto ou HTML."
    }' \
    https://LOO9.com.br/api/sendmail

Exemplo de Resposta (JSON em caso de sucesso):

{
    "status": "success",
    "message": "E-mail enviado com sucesso!",
    "details": {
        "messageId": ""
    }
}

Exemplo de Resposta (JSON em caso de erro):

{
    "status": "error",
    "message": "Falha ao enviar o e-mail.",
    "details": {
        "error": "Descrição detalhada do erro."
    }
}
